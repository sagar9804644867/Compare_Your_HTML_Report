# JMeter Performance Utility

A Streamlit app for two common performance-engineering chores:

1. **JTL → HTML report** — upload one or more JMeter `.jtl` result files and get a
   self-contained, downloadable HTML report (KPI cards, response-time & throughput
   trend charts, and a full per-transaction statistics table).
2. **Compare two HTML reports** — upload a baseline (A) and a current (B) report and
   get a downloadable comparison report that flags regressions and improvements
   per transaction.

## Why it parses JTL in Python (no JMeter binary)

Streamlit Community Cloud has no Java/JMeter install, so `jmeter -g` can't run there.
Instead the app parses the `.jtl` directly with pandas and computes the same
aggregate metrics JMeter reports (average, median, 90/95/99 percentiles, min, max,
error %, throughput, KB/s). Reports this tool generates **embed their statistics as
JSON**, so the comparison step reads exact numbers rather than re-scraping HTML.

## Comparison input compatibility

The comparator extracts stats in this order:

1. Reports generated by this tool (embedded JSON — exact).
2. Any HTML containing an **aggregate-style statistics table** — JMeter *Aggregate
   Report* / *Summary Report* exports, or single-file plugin HTML reports. Column
   headers are matched flexibly (`Label`, `#Samples`, `Average`, `90% Line`, `Error %`,
   `Throughput`, `Received KB/sec`, etc.).

> Note: JMeter's **HTML Dashboard** is a *folder* (its data lives in
> `content/js/dashboard.js`), so a lone `index.html` from it has no inline table to
> read. Generate the report with this tool, or export an Aggregate Report as HTML,
> for the comparison feature.

## Run locally

```bash
pip install -r requirements.txt
streamlit run app.py
```

Open http://localhost:8501.

## Deploy on Streamlit Community Cloud

1. Push this folder to a GitHub repo.
2. Go to https://share.streamlit.io → **New app**.
3. Select the repo/branch, set **Main file path** to `app.py`, and deploy.
   `requirements.txt` is picked up automatically.

## Metric direction

For response-time and error metrics **lower is better**; for throughput **higher is
better**. Changes of **≥10%** are flagged as improvement (green) or regression (red).

## Project layout

```
app.py                      Streamlit UI (two tabs)
requirements.txt
src/
  jtl_parser.py             JTL CSV/XML parsing + statistics engine
  report_generator.py       self-contained HTML report (embeds stats JSON)
  html_comparator.py        extract stats from HTML + compare logic
  comparison_report.py      comparison HTML renderer
samples/                    example .jtl files and generated reports
```

## Notes / limitations

- Percentiles use linear interpolation (numpy), which differs marginally from
  JMeter's exact algorithm but is well within reporting tolerance.
- Trend charts in the JTL report use Chart.js from a CDN; the statistics tables and
  all comparison output work fully offline.
- The JTL parser targets CSV format (JMeter's default). XML JTL is supported on a
  best-effort basis.
